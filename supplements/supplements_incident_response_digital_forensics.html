Session 1 : Threat Landscape & Bases fondamentales :

Trois périmètres à distinguer :

  1. Information Security : Garantir la Confidentialité, l'Intégrité et la Disponibilité (CID ou CIA en anglais) de l'information - quelle que soit sa forme.

  2. IT Security : Protéger les systèmes informatiques et les données contre les accès non autorisés (sous-ensemble de l'InfoSec).

  3. Cybersecurity : Défendre les systèmes connectés contre les attaques malveillantes - focus sur la menace cyber externe.

Vulnérabilité, Menace & Risque :

  • Vulnérabilité : Faiblesse dans un système, un processus, ou une organisation qui peut être exploitée.

    Exemples : patch de sécurité manquant, mot de passe faible, configuration incorrecte.

  • Menace (Threat) : Événement potentiel capable d'exploiter une vulnérabilité et de causer un préjudice.

    Exemples : ransomware, phishing, attaque DDoS, insider malveillant.

  • Risque : Probabilité qu'une menace exploite une vulnérabilité * Impact potentiel.

    RISQUE = VRAISEMBLANCE * IMPACT.

CVE & CVSS - Identifier et scorer les vulnérabilités :

CVE (Common Vulnerabilities and Exposures) :

Identifiant uniqu pour chaque vulnérabilité publique : CVE-ANNÉE-NUMÉRO.

Référentiel maintenu par MITRE, alimenté par les éditeurs et chercheurs.

CVSS - Common Vulnerability Scoring System :

Score de 0 à 10 basé sur : vecteur d'attaque, complexité, impact CIA.

7-8.9 = Élevé. 9-10 = Critique. Exemples : Log4Shell = 10, Spectre = 5.6.

Ressources clés :

https://cve.mitre.org

https://nvd.nist.gov

https://www.cvedetails.com

Catégories de menaces :

Menaces techniques :

  • Malware (ransomware, trojan, wiper).

  • Phishing & spear-phishing.

  • Exploitation de vulnérabilités (RCE, SQLi, XSS).

  • Supply-chain attack.

Menaces organisationnelles :

  • Ingénierie sociale (manipulation humaine).

  • Insider threat (employé malveillant).

  • Fuite de données (accidentelle ou volontaire).

  • APT - Advanced Persistent Threat.

  • Cryptojacking.

Acteurs externes :

  • États et groupes state-sponsored (APT).

  • Crime organisé (ransomware-as-a-service).

  • Hacktivistes (idéologie, protestation).

  • Espionnage industriel (concurrence).

  • Script kiddies (opportunistes).

  • Employé malveillant (insider threat).

  • Employé négligent (erreur humaine).

  • Prestataire / sous-traitant compromis.

  • Ex-employé (accès non révoqué).

  • Partenaire de confiance compromise.

Attack surface, Threat actor & Threat landscape :

Attack surface :

Somme de toutes les surfaces exploitables d'une organisation : interfaces réseau, applications web, emails, postes de travail, accès physiques, employés...

Threat actor / agent :

Entité (personne, groupe, organisation) responsable d'un incident ou capable d'en causer un.

Threat landscape :

Cartographie des menaces pertinentes pour une organisation donnée : qui sont les attaquants probables, par quels vecteurs, pour quels objectifs ?

Chaque secteur a son paysage de menaces propre (finance ≠ santé ≠ logistique).

Session 2 : Hacking Methodology :

Les sept étapes d'une attaque :

  1. Information Gathering

  2. Scanning

  3. Enumeration

  4. Gaining Access (Intrusion)

  5. Privilege Escalation & Lateral Movement

  6. Covering Tracks

  7. Creating Backdoors

→ Chaque étape sera détaillée individuellement dans la suite du cours.

Note : ce schéma est une simplification pédagogique. En réalité, les attaques avancées (APT) ne sont pas linéaires - les étapes se chevauchent et se répètent.

Denial of Service (DoS/DDoS) et Ransomware, sont traités séparément car ce sont des objectifs finaux, pas des étapes systématiques.

  1. Information Gathering (Reconnaissance) :

    Objectif de l'attaquant :

    Collecter un maximum d'informationtions sur la cible AVANT toute interaction directe.

    Passive : OSINT (Open Source Intelligence) :

    • LinkdIn (organigramme, technologies utilisées, emails).

    • WHOIS, DNS records, job postings, GitHub (code source exposé).

    • Shodan (équipements connectés), Google dorks, certificats TLS (crt.sh).

    Active : sondage direct :

    Ping sweep, traceroute, tests de connexion - plus détectable.

  2. Scanning :

    Objectif :

    Identifier les systèmes actifs, ports ouverts, services en écoute, OS.

    Outils courants :

    • Nmap - scanner réseau de référenve (ports, services, versions, OS fingerprinting).

    • Masscan - scanner ultra-rapide pour grands espaces d'adressage.

    • Nesssus / OpenVAS / Qualys - scanners de vulnérabilités.

    Ce que ça révèle :

    • Ports ouverts → services en écoute → versions logicielles → CVEs exploitables.

    • Cartographie du réseau : combien d'hôtes, sous-réseaux, pare-feux ?

  3. Enumeration :

    Objectif :

    Extraire des informations détaillées des services identifiés : comptes, partaes, rôles.

    Cibles typiques :

    • Active Directory (AD) : utilisateurs, groupes, politiques GPO, relations de confiance.

    • SMB : partages réseaux accessibles, fichiers exposés.

    • SNMP : configuration réseau, interfaces, routes.

    • LDAP, HTTP, FTP, SSH : banners, configurations par défaut.

    Résultat :

    L'attaquant connaît maintenant la topologie interne, les comptes actifs et les partages accessibles.

  4. Gaining Access (Intrusion) :

    Méthodes d'exploitation :

    • Exploitation de vulnérabilités : Metasploit, exploits publics (Exploit DB), code custom.

    • Phishing / Spear-phishing → malware (dropper, RAT, ransomware).

    • Credential stuffing / brute force (mots de passe faibles ou leakés).

    • Exploitation d'accès légitimes : VPN, RDP, Citrix mal configurés.

    • Supply-chain : compromission d'un fournisseur de confiance.

    Frameworks d'exploitation :

    • Metasploit (open source) - Cobalt Strike (commercial, très utilisé par APT).

    • Mimikatz - extraction de Credentials depuis la mémoire Windows (LSASS).

  5. Privilege Escalation & Lateral Movement :

    Privilege Escalation (escalade verticale) :

    Passer d'un compte utilisateur standard à Admin/root/SYSTEM.

    Via : exploits kernel, mauvaises configurations, tokens volés, services mal configurés.

    Objectif final souvent : Domain Admin sur Active Directory.

    Lateral Movement (mouvement latéral) :

    Se déplacer d'un système à l'autre dans le réseau interne.

    Techniques : Pass-the-Hash, Pass-the-Ticket, Remote Services (RDP, WMI, PsExec).

    Chaque système compromis devient un nouveau point de départ.

    Distinction clé :

    Escalade = monter en privilèges sur UN système. Latéral = se propager sur D'AUTRES systèmes.

  6. Covering Tracks (Dissimulation) :

    Objectif :

    Effacer les traces de l'intrusion pour rester non détecté et compliquer l'investigation.

    Techniques courantes :

    • Effacement ou modification des journaux (event logs Windows, syslog Linux).

    • Modification des horodatages de fichiers (timestomping).

    • Désactivation des antivirus / EDR

    • Utilisation de comptes légitimes pour masquer l'activité malveillante.

    • Chiffrement des communications C2 (Command & Control) dans du trafic HTTPS légitime.

    Impact pour le forensics :

    Compliquer ou empêcher la reconstitution de la timeline de l'incident.

    → D'où l'importance des logs centralisés (SIEM) et des backups immuables.

  7. Creating Backdoors (Persistance) :

    Objectif :

    Maintenir un accès long terme au système compromis, même après redémarrage ou reset de mot de passe.

    Mécanismes courants :

    • Création de comptes admin cachés dans Active Directory.

    • Tâches planifiées / services Windows malveillants.

    • Modification des DLL système (DLL hijacking) ou des binaires OS.

    • Implants dans le firmware (UEFI rootkits - très difficiles à détecter).

    • Modification des règles de pare-feu ou ACL réseau.

    Lien avec les APT :

    Les APT maintiennent des accès pendant des mois ou des années - phase C2 (Command & Control).

    SolarWinds : backdoor SUNBURST dans une mise à jour logicielle légitime.

Denial of Service (DoS) & DDoS :

Objectif de l'attaque :

Rendre un service inaccessible aux utilisateurs légitimes - impact sur la Disponibilité (D du CID).

Types :

  • Volumétrique : saturer la bande passante (amplification DNS/NTP, exemple : 3,47 Tbps record 2022).

  • Protocole : exploiter les faiblesses TCP/IP (SYN flood, Ping Of Death).

  • Applicatif (Layer 7) : cibler les ressources serveur (HTTP flood, Slowloris).

DDoS = Distributed DoS :

Utilise un botnet (milliers de machines infectées) pour démultiplier l'attaque.

DDoS-as-a-Service disponible sur le dark web pour quelques dizaines d'euros/heure.

Défenses :

CDN avec filtrage DDoS (Cloudflare, Akamai), scrubbing centers, rate limiting.

Ransomware - Mécanisme et impact :

Comment ça fonctionne :

  1. Accès initial (phishing, vuln, credential volé).

  2. Escalade + lateral movement → contrôle de l'AD, accès aux backups.

  3. Exfiltration des données sensibles (double extorsion).

  4. Déploiement du chiffrement sur tout le réseau simultanément.

  5. Demande de rançon + menace de publication des données volées.

Ransomware-as-a-Service (RaaS) :

Groupes comme LockBit, REvil, Conti : fournissent le logiciel + support aux 'affiliés'.

Modèle commercial sophistiqué : partage des revenus, service client, négociation.

Impact réel :

Coût moyen d'un incident ransomware (PME) : > 500k€ - rançon + coûts de récupération.

Kill Chain - Lockheed Martin :

Étape Kill Chain Correspondance 7 étapes Objectif attaquant
1. Reconnaissance Information Gathering + Scanning Identifier la cible et ses faiblesses
2. Weaponization Enumeration Créer ou préparer l'exploit/malware
3. Delivery Gaining Access (vecteur) Livrer l'arme à la cible (email, USB, web)
4. Exploitation Gaining Access (exécution) Déclencher la vulnérabilité
5. Installation Gaining Access + Persistence Installer le malware, établir la persistance
6. C2 (Command & Control) Creating Backdoors Établir un canal de communication sortant
7. Actions on Objectives Exfiltration / Ransomware / Sabotage Atteindre l'objectif final

Session 3 : Incident Response Procedure :

Procédure IR - Vue d'ensemble :

  • Détection

  • Vérification & Qualification

  • Containment

  • Éradication

  • Recovery

  • Lessons Learned

  • Rapport

Ce schéma est votre référence pour toute la session 3. Chaque étape sera détaillée dans la suite du cours.

Parallèle avec le forensics : la collecte de preuves (Digital Forensics) se fait principalement pendant les phases Identification et Containment - avant toute action corrective qui pourrait altérer les preuves.

Détection - Qui voit quoi et comment ?

  • Employés & utilisateurs :

    > 40% des incidents sont d'abord remarqués par un employé non-sécurité. Les campagnes de sensibilisation ont un impact direct.

  • Alertes systèmes & applications :

    Logs serveurs, alertes applicatives, métriques d'infrastructure anormales.

  • Security Monitoring (SOC) :

    • SIEM (Splunk, QRadar, MS Sentinel, ELK) - corrélation de logs multi-sources.

    • EDR/XDR (Tanium, CrowdStrike, MS Defender) - détection comportementale endpoint.

    • NGFW, IDS/IPS, proxies, WAF - alertes réseau.

  • Tiers (CERT, partenaires, clients) :

    Cyber Threat Intelligence (CTI), OSINT, Threat Hunting, disclosure responsable.

Vérification & Qualification :

Question centrale :

Cet événement est-il un incident de sécurité ou une fausse alerte ?

Définition d'un incident de sécurité :

Toute tentative ou acte réussi d'accès non autorisé, utilisation, divulgation, modification ou destruction d'informations ou de systèmes.

→ Impact négatif sur la Confidentialité, l'Intégrité ou la Disponibilité (CID).

Processus de qualification :

  • Recueillir les premiers éléments : logs, alertes, témoignages.

  • Évaluer : est-ce une activité autorisée ? Un test de pénétration planifié ? Un bug ?

  • Si incident confirmé : déclencher la procédure formelle → notifier le Incident Manager.

  • Si non : documenter et archiver pour référence future.

Identification - Comprendre le périmètre :

Objectif :

Délimiter précisément ce qui est affecté, comment l'incident s'est produit et depuis quand.

Actions concrètes :

  • Identifier tous les systèmes, comptes et données impliqués.

  • Analyser les logs réseau, EDR, SIEM pour reconstruire la timeline.

  • Contacter les utilisateurs impactés pour collecter des informations.

  • Identifier le vecteur d'entrée initial (Patient Zéro).

  • Évaluer si l'attaquant est encore actif dans le réseau.

Collecte de preuves forensiques (voir session 5) :

  • Capture mémoire des systèmes critiques AVANT de les isoler ou éteindre.

  • Images disques des systèmes compromis pour analyse ultérieure.

Containment - Limiter la propagation :

Objectif :

Arrêter la propagation de l'incident sans détruire les preuves.

Containment court terme (immédiat) :

  • Isoler les systèmes compromis du réseau (VLAN isolation, déconnexion physique).

  • Bloquer les comptes compromis et révoquer les tokens d'accès.

  • Bloquer les IOC connus sur le pare-feu, proxy, EDR (IPs, domaines, hashs).

Containment long terme :

  • Changer les mots de passe de tous les comptes impactées (AD complet si nécessaire).

  • Vérifier l'intégrité des systèmes non compromis.

  • Patch d'urgence sur la vulnérabilité exploitée.

Attention aux preuves forensiques :

Éteindre un système détruit la mémoire RAM - capturer la mémoire AVANT isolation si possible.

Éradication - Éliminer la menace :

Objectif :

Supprimer complètement la présence de l'attaquant du réseau.

Actions techniques :

  • Supprimer les malwares, fichiers malveillants et outils déposés.

  • Désactiver / supprimer les comptes compromis et backdoors.

  • Réinitialiser tous les credentials potentiellement comprpomis.

Actions organisationnelles :

  • Informer les employés de la menace et des comportements à éviter.

  • Notifier les partenaires et clients impactés.

  • Notifier les autorités compétentes (RGPD, NIS2 - délais légaux !).

Attention : l'éradication n'est pas la recovery :

Ne pas remettre en production avant d'être certain que la menace est éliminée.

Recovery - Reprise d'activité :

Objectif :

Remettre les systèmes en production de manière contrôlée et sécurisée.

Prérequis indispensables :

  • Éradication confirmée : aucune trace de l'attaquant détectée.

  • Vulnérabilités exploitées patchées ou mitigées.

  • Monitoring renforcé activé pour détecter toute réactivation.

Processus de recovery :

  • Restaurer depuis des backups sains et vérifiés (hash de l'image).

  • Prioriser les systèmes critiques pour le business (processus métier clé).

  • Tester avant de remettre en production : accès, intégrité des données, performances.

Recovery ≠ retour à l'état précédent :

C'est l'opportunité de corriger les faiblesses identifiées pendant l'incident.

Lessons Learned & Rapport d'incident :

Lessons Learned (bilan post-incident) :

  • Réunion dans les deux semaines suivant la clôture de l'incident.

  • Pas un exercice de blame - focus sur amélioration des processus.

  • Questions clés : Qu'est-ce qui a fonctionné ? Qu'est-ce qui a échoué ? Comment améliorer ?

  • Résultat : plan d'actions correctives avec responsables et délais.

Structure du rapport d'incident :

  1. Identification : date, nature, systèmes affectés.

  2. Résumé exécutif : ce qui s'est passé, impact, actions menées.

  3. Notification : parties prenantes informées (internes + régulateurs).

  4. Actions : détail de chaque phase IR avec preuves.

  5. Évaluation & enseignements : plan d'amélioration.

  • J+4, 8h00 - Identification :

Mes notes :

Objectifs :

Permettre à l'étudiant de fournir une réponse approprié aux incidents dans l'entreprise, ainnsi qu'entreprendre les démarches forensiques adéquates.

Définitions :

  • La sécurité de l'information est la pratique qui consiste à garantir la confidentialité, l'intégrité et la disponibilité des informations.

  • La sécurité informatique (IT Security) est la pratique visant à protéger les données et les systèmes d'information contre les accès non autorisés. Elle couvre des domaines comme la sécurité des systèmes, la sécurité des réseaux, etc.

  • La cybersécurité est la pratique consistant à défendre les ordinateurs, les serveurs, les réseaux et les données contre les attaques malveillantes.

  • Vulnérabilité : qualité ou état d'un système exposé à la possibilité d'être attaqué ou endommagé, que ce soit physiquement ou logiquement.

  • Menace (Threat) : possibilité d'une tentative malveillante visant à endommager ou perturber un réseau ou un système informatique.

  • Risque (Risk) : situation impliquant une exposition à un danger ou à une menace.

  • Vulnérabilité zero-day : une vulnérabilité dans un système ou un appareil qui est inconnue des personnes chargées de la corriger (et donc non encore corrigée).

  • La surface d'attaque correspond à l'ensemble des vulnérabilités exploitables pour mener une attaque de sécurité, qu'elles soient physiques ou numériques.

  • Un acteur de la menace (Threat actor/agent) est une entité (personne, groupe ou organisation) responsable, en partie ou totalement, d'un incident ayant un impact ou pouvant impacter la sécurité d'une organisation.

  • Le paysage des menaces (Threat landscape) désigne l'ensemble des caractéristiques, des méthodes d'attaque et des objectifs des différents types de menaces auxquels une organisation est exposée.

  • La criminalistique (Forensics) regroupe l'ensemble des techniques et méthodes scientifiques utilisées pour détecter, analyser et prouver des faits liés à une activité criminelle.

  • L'informatique légale (Digital Forensics) consiste à collecter, préserver, analyser et présenter des preuves numériques (ordinateurs, réseaux, téléphones, etc.) dans le cadre d'une enquête.

Exemples de vulnérabilités : Mitre Common Vulnerabilities and Exposures (CVE).

La Security Risk Matrix (Matrice des risques de sécurité) est un outil qui permet d'évaluer et de prioriser les risques en fonction de leur probabilité et de leur impact.

Risque = Probabilité x Impact : Le risque est le produit de la probabilité (likelihood) qu'un événement se produise et de son impact (les conséquences) s'il se réalise.

Exemples de menaces :

Nom de la menace Rôle/Description
Attacks of IoT Devices (Attaques des appareils IoT) Ces attaques exploitent les failles des objets connectés souvent mal sécurisés pour accéder à un réseau ou lancer d'autres attaques.
Phishing (Hameçonnage) Cette attaque consiste à tromper un utilisateur pour qu'il divulgue des informations sensibles comme des mots de passe ou des données bancaires.
Spear Phishing (Hameçonnage ciblé) Il s'agit d'une forme de phishing personnalisée visant une personne ou une organisation spécifique pour augmenter les chances de succès.
MITM - Man-in-the-Middle (Attaque de l'homme du milieu) L'attaquant intercepte et peut modifier les communications entre deux parties sans qu'elles s'en rendent compte.
Trojans (Chevaux de Troie) Ce type de malware se présente comme un logiciel légitime afin d'inciter l'utilisateur à l'installer et compromettre le système.
Ransomware (Rançongiciel) Ce logiciel malveillant chiffre les données de la victime et exige une rançon pour en rétablir l'accès.
Denial of Service attack - DoS (Attaque par déni de service) Cette attaque surcharge un système ou un service pour le rendre indisponible aux utilisateurs légitimes.
DNS attack (Attaque DNS) Cette attaque vise à détourner ou falsifier les requêtes DNS afin de rediriger les utilisateurs vers de faux sites.
Cross-Site Scripting - XSS (Script intersite) Elle permet d'injecter des scripts malveillants dans des pages web consultées par d'autres utilisateurs.
Buffer overflow (Dépassement de tampon) Cette vulnérabilité est exploitée pour écrire au-delà de la mémoire allouée et exécuter du code malveillant.
Shortage of Skilled Cybersecurity Professionals (Pénurie de professionnels qualifiés en cybersécurité) Le manque d'experts en cybersécurité augmente le risque d'erreurs, de mauvaises configurations et d'attaques réussies.
Supply Chain attack (Attaque de la chaîne d'approvisionnement) L'attaquant compromet un fournisseur ou un partenaire pour atteindre indirectement la cible principale.
Insider threat (Menace interne) Cette menace provient d'un utilisateur interne qui abuse de ses accès ou agit par négligence.
Malware propagation (Propagation de logiciels malveillants) Il s'agit de la diffusion d'un malware à travers un système ou un réseau pour infecter un maximum de machines.
Social Engineering (Ingénierie sociale) Cette technique manipule psychologiquement les individus pour les inciter à révéler des informations ou à effectuer des actions risquées.
Data leakage (Fuite de données) Cela correspond à la divulgation non autorisée d'informations sensibles, volontaire ou accidentelle.
Remote Code Execution - RCE (Exécution de code à distance) Cette vulnérabilité permet à un attaquant d'exécuter du code sur un système distant sans autorisation.
Advanced Persistent Threats - APT (Menaces persistantes avancées) Ce sont des attaques sophistiquées et prolongées visant à infiltrer discrètement un système sur le long terme.
Cryptojacking Cette attaque utilise les ressources d'un système à l'insu de l'utilisateur pour miner des cryptomonnaies.
Distributed Denial of Service - DDoS (Attaque par déni de service distribué) Elle consiste à submerger un service avec du trafic provenant de multiples sources pour le rendre indisponible.
Cross-Site Request Forgery - CSRF (Falsification de requête intersite) Cette attaque force un utilisateur authentifié à exécuter une action à son insu sur une application web.
Privilege Escalation (Élévation de privilèges) L'attaquant obtient des droits d'accès plus élevés que ceux initialement autorisés.
Lateral movement (Mouvement latéral) Après une intrusion, l'attaquant se déplace dans le réseau pour étendre son accès à d'autres systèmes.
Memory Corruption (Corruption de mémoire) Cette attaque exploite des erreurs de gestion mémoire pour altérer le fonctionnement d'un programme.

Sources des cybermenaces :

Nom de la source Rôle/Description
State-sponsored (Étatiques / parrainés par un État) Ces acteurs sont soutenus par des gouvernements et mènent des cyberattaques pour l'espionnage, l'influence géopolitique ou le sabotage.
Cyber terrorists Ils utilisent les cyberattaques pour semer la peur, perturber des infrastructures et promouvoir une idéologie.
Industrial spies (Espions industriels) Ils cherchent à voler des secrets commerciaux ou des informations stratégiques pour obtenir un avantage concurrentiel.
Organized crime groups (Groupes criminels organisés) Ces groupes mènent des activités cybercriminelles, souvent pour un gain financier comme la fraude ou le ransomware.
Hackers Ce terme désigne des individus qui exploitent des failles informatiques avec des intentions qui peuvent être malveillantes ou non.
Hacktivists Ils réalisent des cyberattaques pour défendre des causes politiques ou sociales.
Malicious insider (Menace interne malveillante) Il s'agit d'un employé ou d'un utilisateur interne qui abuse volontairement de ses accès pour nuire à l'organisation.
Cyber espionage Activité consistant à infiltrer des systèmes pour collecter discrètement des informations sensibles.
Human error (Erreur humaine) Les erreurs ou négligences des utilisateurs peuvent involontairement créer des failles de sécurité ou provoquer des incidents.
